# DTRD 双层训练中的四个 Loss 详解

## 背景：为什么是「双层」

DTRD 要解决**延迟奖励**：一整条轨迹只有最后一步有奖励（sparse rtg）。DT 靠 return-to-go 条件工作，但每步 rtg 全一样（都等于总回报），模型学不到「哪一步动作重要」。

DTRD 的思路：训一个**重分配网络**，把那笔终点奖励「打散」重新分到每一步 → 每步有了不同的 rtg → 策略能学到细粒度。

问题：重分配网络怎么训？没有 ground truth 告诉它「这步该给多少奖励」。DTRD 的答案是**元学习（bilevel）**：

> 好的重分配 = 用它训出的策略，在**没见过的验证集**上表现好。

于是两层：

- **下层（内层）**：固定重分配网，用重分配后的 rtg 训策略网（DT）。
- **上层（外层）**：调重分配网的参数，使得「训出来的策略在 val 上 loss 低」。

两层各有独立优化器，**没有合并 loss**。

---

## 记号

| 符号 | 含义 |
|------|------|
| θ | 重分配网参数（上层） |
| φ | 策略网（DT）参数（下层） |
| `r_t` / `r_v` | train / val 批的 sparse rtg |
| `r_m` | 重分配后的 rtg = `sparse_rtg − 学到的重分配回报` |

---

## 四个 Loss 逐个讲

### ① `policy/train_loss` — 下层第一步

```python
_, loss = model(x_t, y_t, y_t, r_t_m, t_t); loss.mean()
```

**是什么**：策略网在 **train 批**上的动作预测交叉熵。给定（状态、重分配后的 rtg、时间步），预测该走哪个动作，和数据集里真实动作比。

**更新谁**：策略网 φ。标准模仿学习的 loss。

**怎么看**：应稳定下降。这是模型「学会模仿数据」的直接信号。

---

### ② `policy/val_loss` — 下层第二步

```python
_, loss = model(x_v, y_v, y_v, r_v_m, t_v); loss.mean()
```

**是什么**：同样的策略 CE，但在 **val 批**上，再对 φ 走一次 SGD。

**为什么还要在 val 上训策略**：这是官方 DTRD 的设计 —— 策略每个 iter 走两步（train 批 + val 批），让策略把 train/val 都用上做模仿。注意这里用的是 `r_v_m`（**detach 过**的重分配 rtg），所以这步**不**给重分配网传梯度，纯更新策略。

**怎么看**：和 ① 一起看。

- 两个都降、贴得近 → 健康，泛化好
- train 降、val 不降或反升 → 策略过拟合训练集

---

### ③ `redistribute/meta_val_loss` — 上层元目标（核心）

```python
val_loss, _ = gradient_appro.redistribute_step(...)
```

**是什么**：这是整个方法的灵魂。它衡量的是——

> 先假想「用当前重分配训练策略走一小步」得到临时策略 φ'，再看 **φ' 在 val 批上的 CE**。

这个 val CE 就是 `meta_val_loss`。它是重分配网 θ 的**元目标**：θ 变好 → 训出的 φ' 在 val 上更准 → 这个 loss 降。

**更新谁**：重分配网 θ。梯度通过 DARTS 二阶近似（Hessian-向量积）从「val 上的策略表现」一路传回 θ。这就是 `GradientApproximate` 那套 φ'、φ+、φ− 有限差分在算的东西。

**怎么看**：**最该盯的曲线**。持续下降 = 重分配网确实在学「怎么分奖励才能让策略泛化好」。如果它不降 → 重分配没在起作用，方法退化成普通 DT。

---

### ④ `redistribute/reg_loss` — 轨迹守恒正则

```python
reg_loss = redistribute_sum_square.item()
# = trajectory_lamb * (Σ重分配step_reward * traj_len/context_len − head_rtg)² / batch
```

**是什么**：约束项。要求「重分配出来的每步奖励，按比例加总后要**约等于原始轨迹总回报**」。即打散可以，但**总量得守恒**，不能凭空造/毁奖励。

**为什么要**：没有它，重分配网可以作弊——把奖励任意放大或清零去压低 meta_val_loss，得到无意义的 rtg。这项把它拉回「总量 = 真实回报」。`trajectory_lamb`（默认 0.01）是它的权重。

**怎么看**：应保持小而稳。

- 稳定在低位 → 守恒约束满足，重分配合理
- 爆涨 → 重分配网跑飞了，rtg 失真（常伴随策略崩）

---

## 关系图

```
       ┌─────────── 上层：训重分配网 θ ───────────┐
       │  ③ meta_val_loss  (φ' 在 val 上的 CE)    │  ← θ 的主目标，要降
       │  ④ reg_loss       (总量守恒约束)          │  ← 保持小而稳
       └──────────────────────────────────────────┘
                          │ θ 决定怎么打散奖励
                          ▼
       ┌─────────── 下层：训策略网 φ ─────────────┐
       │  ① policy/train_loss (train 批 CE)       │  ← 要降
       │  ② policy/val_loss   (val 批 CE)         │  ← 跟①贴近=不过拟合
       └──────────────────────────────────────────┘
```

**没有加权总和**：①② 更新 φ（策略优化器），③④ 的梯度更新 θ（重分配优化器），两条链各走各的。

---

## 健康训练长这样

- ①② 一起平滑下降且贴近
- ③ 稳定下降（重分配在帮忙）
- ④ 低位平稳
- → 最终 `eval/return` 上升

---

## 排查用法

哪条异常直接指向哪层出问题：

| 现象 | 诊断 |
|------|------|
| ③ 不降但 ① 降 | 重分配没学到东西，分数上不去的锅在上层 |
| ① 降 ② 不降 | 策略过拟合训练集 |
| ④ 爆涨 | 重分配失控，rtg 失真 |
